# DRYAD 충남 AI 수목관리 의사결정 대시보드

> **충남 1,602개 수목관리 대상의 위험도를 6개 공공데이터로 자동 산출하여, 지자체 공원녹지·산림녹지 담당자의 "어디를 먼저 관리할지" 의사결정을 지원하는 웹 기반 SaaS.**

제14회 충남 공공데이터·AI활용 창업경진대회 출품작 · 드라이어드(전 나무잇)

---

## 1. 왜 만들었나 — 문제 정의

충남 시군의 공원·가로수·도시숲 관리는 다음 한계를 안고 있다.

| 구조적 한계 | 결과 |
|---|---|
| 아날로그·수기 중심 운영, 담당자별 데이터 분산 | 데이터 기반 **선제적 관리 불가** |
| 비표준화 보고 체계, 현장 디지털 기록 유실 | 조직 내 **업무 효율성 저하** |
| 객관적 성과 지표 부재, 이해관계자 설득 약화 | **예산 확보·정책 반영 실패** |

이 결과 가로수 고사율이 높고, 적기 조치 기회를 놓치며, 사후 관리 예산 낭비가 누적된다. 한편 산림·수목은 NDC 달성의 핵심 탄소 자산이지만 매년 약 6,000만 톤의 흡수량 격차가 발생할 것으로 추정된다.

**충남 단위로 좁히면**: 15개 시군 공원녹지과·산림녹지과 담당자가 매일 부딪치는 문제 — "이번 주에 어느 구역을 먼저 점검해야 하나?" — 에 대한 객관적 근거가 없다.

---

## 2. 솔루션 — 한 줄 정의

> **공공데이터(산림청·기상청·농진청·KOSIS·표준데이터셋)를 자동으로 통합·분석하여, 사이트별 위험도와 관리 우선순위를 지도·우선순위·시군별 히트맵으로 시각화하고, 행정용 PDF 보고서로 즉시 출력 가능한 의사결정 대시보드.**

핵심 가치 명제 3가지:

1. **공공데이터 + 행정 데이터 융합** — 외부 5개 출처 + 사이트 자체 메타데이터 + 지자체 업로드 CSV(새올·신문고)를 단일 분석 레이어로 결합
2. **정직한 위험도 산정** — 모든 raw 입력과 변환 산식을 인앱 모달과 SCORING.md로 공개. 시뮬레이션·추정값은 명시적으로 라벨링
3. **의사결정 산출물 직접 생성** — 시연 가능한 웹앱 + 단일 클릭 PDF 보고서 + 시군별 6요인 히트맵

## 3. 누가 쓰나 — 타깃 사용자

- **1차 (의사결정)**: 충남 15개 시군의 공원녹지과·산림녹지과·푸른도시과·정원도시과 담당 공무원
- **2차 (현장 협력)**: 나무병원·수목관리 용역업체·나무의사
- **3차 (정책·감사)**: 충남도청 산림녹지정책관, 감사관실, 충남도의회 산림 관련 위원회

특히 충남 시군 공원녹지과는 다음 작업에 직접 활용 가능:

- 분기별 정기 점검 우선순위 수립
- 차년도 예산 편성 근거 자료 (PDF 보고서 출력)
- 민원·재난 대응 시 즉시 위험 사이트 식별
- 감사 대응 시 관리이력·근거 데이터 추출
- 시군 간 위험도·관리 현황 비교 (시군별 히트맵)

## 4. 무엇을 보여주나 — 핵심 기능

### 4.1 통합 대시보드
- **충남 1,602개 사이트**가 한 지도에 클러스터링되어 표시
- 시군별 마커 묶음의 색이 그 시군 **최고 위험등급**을 즉시 반영
- 좌측 패널: 5종 요약 카드 (전체 / A등급 / 주간 점검 권장 / 산불위험 연계 / 예상 관리 예산 / **연간 탄소 흡수량**)
- 좌측 하단: 위험도 정렬 + 시군 필터 적용된 우선순위 테이블

### 4.2 시군별 분석 (히트맵 탭)
- 충남 15개 시군 × 6요인 매트릭스
- 셀 색상으로 시군별 약점을 즉시 식별 (예: "천안=관리공백, 보령=산불, 태안=기상")
- 셀 클릭 시 해당 시군으로 자동 필터링
- 각 컬럼 헤더에 데이터 출처 배지(`실데이터`/`추정`/`시뮬레이션`)

### 4.3 사이트 상세 패널 (우측)
- 6요인 위험도 막대 그래프 + 출처 배지
- AI 규칙 기반 추천 조치 (1·2순위 + 예상 비용)
- 관리이력 누적 (모달로 추가)
- 민원 누적 (CSV 업로드 시 자동 매칭)
- **PDF 보고서 1클릭 다운로드** (A4 1~2페이지)

### 4.4 검색·필터
- 대상명·주소·수종·관리기관·사이트ID 부분 일치 검색
- 시군구·유형·위험등급·출처·민원 다중 필터

### 4.5 CSV 업로드 (2종)
- **사이트 CSV**: 충남 공공데이터 표준 (소나무숲·가로수길·도시공원) 자동 스키마 인식 + 시군 매핑 + 위험도 자동 계산
- **민원 CSV**: 새올·국민신문고 export 형식. 좌표 기반 500m 이내 가장 가까운 사이트에 자동 귀속. 누적 시 위험도 가산점 자동 반영

### 4.6 산식·출처 모달
- 6요인별 raw 데이터·변환 산식·정직성 분류 전체 공개
- 클릭 가능한 원본 데이터 링크 (data.go.kr·KOSIS 등)
- 심사·감사·MOU 시 신뢰 확보 도구

### 4.7 데이터 신선도 모달
- 6요인 각각의 마지막 갱신 시각·출처·주기·단위
- "산불위험: 3시간 전 갱신 / 기상: 5/23 08시 발표"

### 4.8 PDF 보고서
- 1클릭 다운로드, 파일명 `DRYAD_{사이트명}_{날짜}.pdf`
- 포함: 대상 기본정보 + 위험도 평가 + 6요인 분석 + AI 추천 조치 + 관리이력 + 민원·신고 + 데이터 출처 푸터
- 공무원이 회의 자료·결재 문서로 즉시 활용

## 6. AI는 어떻게 쓰이나 — 위험도 산정

### 6.1 종합 위험도 산식 (기획서 9.1)

```
종합 위험도 =
  기상 스트레스       × 0.25
+ 산불 위험           × 0.20
+ 식생·수종 취약성   × 0.15
+ 토양·지형          × 0.10
+ 관리공백·노후도    × 0.20
+ 피해 이력          × 0.10
+ 민원 누적 가산점   (사이트별 최대 +25)
```

### 6.2 위험등급

| 점수 | 등급 | 의미 |
|---|---|---|
| 80~100 | A | 즉시 점검 필요 |
| 60~79 | B | 단기 점검 권장 |
| 40~59 | C | 정기 관리 |
| 0~39 | D | 낮은 위험 |

### 6.3 6요인 각각의 산정 방법

| 요인 | 데이터 | 산정 방법 |
|---|---|---|
| 기상 스트레스 | 기상청 단기예보 | 임계값 가산식 (33℃↑·40%↓·8m/s↑·5mm↓) → 0~100 정규화 |
| 산불 위험 | 산림청 API | `meanavg` 직접 사용 (변환 없음) |
| 식생·수종 취약성 | 수종 매트릭스 v0.1 | 100여 수종별 점수 (소나무 85, 은행나무 15 등) · 복합수종은 max() |
| 토양·지형 | 농진청 토양도 V2 | 심토토성 + 자갈 + 경사 가중평균 (0.4/0.3/0.3) |
| 관리공백·노후도 | CSV 지정고시일/식재년도 (proxy) | 경과년수 5단계 컷오프 (50년+ 90점 ~ 5년 미만 25점) |
| 피해 이력 | KOSIS 산림청 통계 | 시도 단위 연간 발생건수·피해면적 정규화 |

### 6.4 AI 추천 조치 (규칙 기반)

각 위험요인의 임계 초과 패턴에 따라 자동 조치 추천:

- 관리공백 ≥ 70 → 현장점검·전정
- 산불위험 ≥ 75 → 현장점검·제거 (고사가지·낙엽 제거)
- 기상 스트레스 ≥ 70 → 관수 점검
- 식생 ≥ 65 → 정기 방제 검토
- 피해이력 ≥ 55 → 안전점검
- 토양 ≥ 55 → 토양개량

추천에 예상 비용(전정 250만원·방제 350만원 등)을 자동 산출하여 예산 편성 근거 제공.

### 6.5 정직성 분류

각 요인의 데이터 출처를 3단계로 명시 라벨링:

- 🟢 **PUBLIC**: 외부 기관이 발행한 raw 공공데이터 (산불·기상·토양·피해이력)
- 🟦 **PLAN**: 사용자 기획서에 명시된 가중치·등급 컷오프
- 🟧 **MVP**: 운영 단계 캘리브레이션 필요한 기본값 (수종 매트릭스·점수 변환 산식)

전체 산식은 [SCORING.md](SCORING.md) 및 인앱 "산식·출처 보기" 모달에 공개.

## 8. 기술 스택

| 영역 | 기술 |
|---|---|
| 프론트엔드 | React 18 + Vite 5 |
| 지도 | Leaflet + react-leaflet + react-leaflet-cluster (OpenStreetMap 타일) |
| UI | Tailwind CSS · Pretendard 한국어 폰트 |
| 데이터 처리 | Node.js ETL 스크립트 (CSV 파싱 + EUC-KR 디코딩 + API 통합) |
| 보고서 | jsPDF + html2canvas (한국어 PDF 생성) |
| 배포 | 정적 호스팅 (Vercel/Netlify 대응) |

소스 구조:
```
dryad-mvp/
├── data/
│   ├── raw/              # 다운로드한 공공데이터 CSV 원본
│   └── external/         # API 호출 결과 캐시 (JSON)
├── scripts/
│   ├── etl.mjs           # CSV → sites_real.json 변환
│   └── fetchExternal.mjs # 4개 OpenAPI 자동 수집
└── src/
    ├── components/       # 12개 React 컴포넌트
    ├── lib/              # 위험 산식·수종 매트릭스·탄소·CSV 파서
    └── data/             # 통합 데이터 로더
```

## 9. 현재 한계와 솔직한 분류

### 9.1 정직성 라벨

| 분류 | 비중 | 영역 |
|---|---|---|
| 🟢 Raw 입력 데이터 | 6요인 중 5요인의 input | 100% 공공데이터·공식 통계 |
| 🟦 가중치·등급 컷오프 | 종합 산식 | 기획서 9.1·9.2 인용 |
| 🟧 Raw → 점수 변환 산식 | 5개 요인 | MVP 단계 합리적 기본값 (운영 시 캘리브레이션 필요) |

### 9.2 명시적 한계

- **사이트 단위 (단목 단위 아님)**: 가로수길 전체·공원 전체가 하나의 객체. 개별 수목 단위 관리는 IR덱 로드맵 (수목 EMR, 2027)
- **AI는 규칙 기반**: 현재 학습 모델 없음. IR덱의 처방적 AI (CEI·SDR·WSI·CSM·Poisson 산불 예측)는 운영 데이터 누적 후 구축
- **민원·관리이력 영속 저장 없음**: 브라우저 메모리 기반. 운영 시 Supabase·자체 DB 연계
- **위성·드론·LiDAR 미사용**: 도시 사이트 위주라 영향 적음. IR덱 로드맵 (2028+)
- **곡교천 365주 같은 단목 데이터 미반영**: 현 MVP는 표준 공공데이터셋만 처리. 단목 데이터는 별도 CSV 업로드 또는 운영 단계 통합
