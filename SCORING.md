# DRYAD 충남 MVP — 위험도 산정 방법론

본 문서는 6요인 종합 위험도의 raw 데이터·변환 산식·정직성 분류를 모두 공개합니다. 발표·심사 자료로 그대로 사용 가능합니다.

---

## 종합 위험도 산식

```
종합 위험도 (0~100)
= 기상 스트레스    × 0.25
+ 산불 위험        × 0.20
+ 식생·수종 취약성 × 0.15
+ 토양·지형        × 0.10
+ 관리공백·노후도  × 0.20
+ 피해 이력        × 0.10
```

| 점수 | 등급 | 의미 |
|---|---|---|
| 80~100 | **A** | 즉시 점검 필요 |
| 60~79 | **B** | 단기 점검 권장 |
| 40~59 | **C** | 정기 관리 |
| 0~39 | **D** | 낮은 위험 |

**출처**: 가중치·등급 컷오프는 모두 [드라이어드 충남 MVP 기획서](../dryad_chungnam_mvp_planning.md) 9.1 / 9.2 절에 정의된 값을 그대로 사용.

---

## 표기 범례

각 산식의 항목은 다음 3가지 출처로 분류됩니다:

| 표기 | 의미 |
|---|---|
| 🟢 **PUBLIC** | 외부 기관이 발행한 raw 공공데이터 또는 공식 통계. 손대지 않고 그대로 사용 |
| 🟦 **PLAN** | 사용자 기획서에 명시된 임계값·가중치 |
| 🟧 **MVP** | 현 MVP 단계의 합리적 기본값. 운영 시 전문가 자문으로 캘리브레이션 필요 |

## 1. 기상 스트레스 점수 (weather_stress_score)

### Raw 데이터 — 🟢 PUBLIC
- **출처**: [기상청_단기예보 조회서비스](https://www.data.go.kr/data/15084084/openapi.do)
- **갱신**: 매일 3시간 간격 (02, 05, 08, 11, 14, 17, 20, 23시 발표)
- **호출**: 충남 15개 시군의 격자 좌표(nx, ny)별 향후 48~72시간 예보
- **취득 컬럼**: TMP(기온) / TMX(최고기온) / REH(습도) / WSD(풍속) / PCP(강수량)

### 집계 — 🟧 MVP
시군별 예보 전체에서 다음 4개 통계량 추출:
- `maxTemp` = max(TMP, TMX)
- `minHumidity` = min(REH)
- `maxWind` = max(WSD)
- `totalRain` = sum(PCP)

### 점수 산식 — 🟦 PLAN (기준값) + 🟧 MVP (정규화)

```
raw =
  (maxTemp     ≥ 33°C   → 25 |  ≥ 30   → 15 |  ≥ 27   →  8 |  else 0)   🟦
+ (minHumidity ≤ 40%    → 20 |  ≤ 50   → 10 |  else 0)                   🟦
+ (maxWind     ≥ 8 m/s  → 20 |  ≥ 5    → 10 |  else 0)                   🟦
+ (totalRain  ≤ 5 mm    → 25 |  ≤ 10   → 10 |  else 0)                   🟦

→ raw 범위 0~90
score = round(raw / 90 × 100)  → 0~100 정규화                            🟧
```

- 🟦 임계값(33도, 40%, 8m/s, 5mm)과 가산점(+25, +20)은 기획서 9.3에서 인용
- 🟧 90→100 정규화 방식과 단계 분할(예: 33도/30도/27도 3단계)은 MVP 결정

## 2. 산불 위험 점수 (fire_risk_score)

### Raw 데이터 — 🟢 PUBLIC
- **출처**: [산림청 국립산림과학원 산불위험예보정보](https://www.data.go.kr/data/15084817/openapi.do)
- **엔드포인트**: `forestPointListSigunguSearchV2`
- **갱신**: 매일 8회 (3시간 간격)
- **취득 컬럼**: `meanavg` (시군구별 평균 산불위험지수, 산림청 표준 0~100 척도)

### 점수 산식 — 🟢 PUBLIC

```
score = meanavg   (변환 없음, 산림청 공식 지수 그대로)
```

**해당 요인은 100% 산림청 raw 데이터.** 자체 가공 없음.

| 침엽수 | 잣나무 | 70 | 재선충 취약 |
| 침엽수 | 메타세쿼이아 | 20 | 강함 |
| 활엽수 (가로수 흔함) | 은행나무 | 15 | 거의 무병 |
| 활엽수 | 느티나무 | 45 | 노거수 도복 위험 |
| 활엽수 | 벚나무·왕벚 | 60 | 흰가루병·사철혹병 |
| 활엽수 | 이팝나무 | 25 | 강함 |
| 활엽수 | 단풍나무류 | 40 | 흰가루병 |
| 활엽수 | 아카시아 | 65 | 도복 빈발 |

**복합 수종 처리** (예: "벚나무+이팝나무"):
```
score = max(개별 수종 점수)  → 가장 취약한 수종 기준 (보수적)
```

**근거**:

- 도시수목 관리 일반론 (도복·전정·노거수 안전)

**한계**: 매트릭스 자체는 검증된 학술표 아닌 MVP 합리적 기본값. **운영 시 산림청·나무의사 자문으로 v1.0 캘리브레이션 필요.**

## 4. 토양·지형 점수 (soil_score)

### Raw 데이터 — 🟢 PUBLIC
- **출처**: [농촌진흥청 토양도 기반 토양특성 단면정보 V2](https://www.data.go.kr/data/15144105/openapi.do)
- **엔드포인트**: `/SoilEnviron/SoilCharacSctnn/V2/getSoilCharacterSctnn`
- **호출 방식**: 시군별 대표 PNU 1건씩 (15회 호출). 시청·군청 소재 법정동 1번지, 도심 데이터 부재 시 외곽 농촌으로 대체
- **취득 코드**: Deepsoil_Qlt_Cd / Deepsoil_Ston_Cd / Soilslope_Cd

### 점수 산식 — 🟧 MVP

**코드 → 점수 매핑**:

| 심토토성 코드 | 의미 | 점수 |
|---|---|---|
| 01 | 사질 | 70 (보수력 낮음) |
| 02 | 사양질 | 40 |
| 03 | 미사사양질 | 35 |
| 04 | 식양질 | 20 (가장 유리) |
| 05 | 미사식양질 | 25 |
| 06 | 식질 | 60 (배수 불량) |

| 자갈함량 코드 | 의미 | 점수 |
|---|---|---|
| 01 | 없음 (0~15%) | 20 |
| 02 | 있음 (15~35%) | 50 |
| 03 | 심함 (35%+) | 80 |

| 경사도 코드 | 의미 | 점수 |
|---|---|---|
| 01 | 0~2% | 20 |
| 02 | 2~7% | 25 |
| 03 | 7~15% | 40 |
| 04 | 15~30% | 60 |
| 05 | 30~60% | 80 |
| 06 | 60~100% | 95 |

**가중 평균**:
```
soil_score = 토성 × 0.4 + 자갈 × 0.3 + 경사 × 0.3
```

**근거**: 식양질·미사식양질이 수목 생육에 유리, 사질·식질은 보수·배수에 문제, 자갈이 많을수록 뿌리 발달 저해, 경사가 클수록 침식 및 관리 어려움 — 토양학·임업학 일반론.

**한계**:
- 매트릭스 값은 MVP 기본값. 수목 종별 적합성 모델로 확장 가능
- 시군당 PNU 1건이라 시군 내 지점간 토양 차이는 반영 안 됨 (VWorld 지오코딩 연계 시 사이트별 정밀화 가능)

## 5. 관리공백·노후도 점수 (management_gap_score)

### Raw 데이터 — 🟢 PUBLIC + 🟧 MVP (해석)

- **출처**:
  - 도시공원 CSV의 `지정고시일` (1968~2020)
  - 가로수길 CSV의 `식재년도` (1980~2022)
  - 소나무숲: 데이터 없음 → 기본값 50

### 변환

```
경과년수 = TODAY − 지정고시일/식재년도        🟢
```

**점수 매핑** — 🟧 MVP

| 경과년수 | 점수 |
|---|---|
| 50년+ (1976년 이전 조성) | 90 |
| 30~50년 (1976~1996) | 75 |
| 15~30년 (1996~2011) | 60 |
| 5~15년 (2011~2021) | 45 |
| 5년 미만 (2021~) | 25 |
| 미상 (소나무숲 등) | 50 |

**중요한 한계 — 🟧 MVP 해석**:

기획서 9.5는 본래 **"마지막 관리이력 후 경과기간"** 기준. 우리는 실제 관리이력 DB 미보유 상태에서 **"조성·식재 후 경과"를 proxy**로 사용한다.

> 예: 1968년 조성 공원이 매년 정기 관리 받으면 실 관리공백은 작음. 그러나 우리 점수는 90점.
>
> 50년 된 공원이라는 사실 자체가 노후 시설·노거수 안전점검 필요도와 상관관계 있다는 가정 하에 사용.

**정밀화 경로**: 충남 지자체 관리이력 DB와 연계 → "실 관리공백"으로 전환 가능.

UI에는 "관리공백 / 노후도"로 표기, 출처 배지는 `추정`으로 명시.

## 6. 피해 이력 점수 (damage_history_score)

### Raw 데이터 — 🟢 PUBLIC
- **출처**: [KOSIS 국가통계포털 — 시도별 산불발생 현황](https://kosis.kr/statHtml/statHtml.do?orgId=136&tblId=DT_136N_010002)
- **원 출처**: 산림청 산불통계
- **취득 값** (충남 시도, 2016~2025 10년 평균):
  - 발생 건수: 43.0건/년
  - 피해 면적: 284.16 ha/년

### Granularity 한계
- KOSIS 시도 단위 통계 → 충남 = 1줄
- **15개 시군에 동일값 적용**. 시군별 차별화 부재.
- 정밀화 경로: 산림청 산림통계연보 시군별 데이터 연계 (PDF 수동 입력 또는 산림청 시군 통계 API 안정화 후)

### 점수 산식 — 🟧 MVP

```
freq_score = min(100, count_yr / 100 × 100)      → 43 → 43점        🟧
area_score = min(100, area_ha_yr / 1000 × 100)   → 284.16 → 28.4점  🟧
damage_history_score = freq × 0.6 + area × 0.4   → 37점             🟧
```

- 🟧 정규화 기준 (100건/년=100점, 1000ha/년=100점) 은 MVP 결정
- 🟧 빈도/면적 가중치 (0.6/0.4) 도 MVP 결정

**한계**: 시도 단위 통계라 충남 15개 시군이 모두 같은 37점. 시군별 분해는 산림청 통계연보 또는 OpenAPI 게이트웨이 안정화 후 가능.

## 정직성 요약

| 분류 | 비중 | 비고 |
|---|---|---|
| 🟢 Raw 입력 데이터 | **공공데이터 / 공식 통계 100%** | 산림청·기상청·농진청·KOSIS·표준데이터셋 |
| 🟦 가중치·등급 컷오프 | 사용자 기획서 (외부 근거 없음) | 합리적 기본값 |

**시연·발표 권장 멘트**:

> "**6요인의 raw 입력은 100% 공공데이터·공식 통계입니다.** 산불·기상·토양은 실시간 OpenAPI, 피해이력은 KOSIS 공식 통계, 사이트 데이터는 표준데이터셋 16개를 받았습니다.

## 향후 캘리브레이션 경로

1. **수종 매트릭스 v1.0**: 산림청 나무병원 협회·나무의사 자문으로 100+개 수종의 점수 재조정

3. **관리공백 → 실 관리이력**: 충남 지자체 관리이력 DB와 API 연동 (Tier 3)
4. **피해 이력 시군별**: 산림청 산불통계 OpenAPI 안정화 또는 산림통계연보 시군별 분해
5. **추천 조치 단가**: 충남 시군 공원녹지과 표준 단가 기준 교체
6. **가중치 재조정**: 충남 시범 운영 후 시군 담당자 피드백 기반 재조정
